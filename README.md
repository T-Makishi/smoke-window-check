# 排煙窓かんたん事前チェック

排煙窓の修理・点検前に、お客様が問診と写真・動画を選択し、修理費用の目安と現地調査費の条件を確認できるWebアプリです。

現在は画面、概算、端末内保存、印刷、メール作成まで実装したプロトタイプです。問診・写真・動画を会社へ安全に自動送信する本番バックエンドはこれから接続します。この状態を正式な受付には使用しないでください。実運用の構成は `PRODUCTION_DESIGN.md`、作業手順は `START_GUIDE.md` を参照してください。

UIはデザインA「信頼感・安心感のあるベーシックデザイン」を採用しています。ダークグリーン、ネイビー、白を基調とし、料金案内だけを淡い黄色で明示しています。最大表示幅は480pxで、iPhone 13と320px幅へ対応しています。主要アイコンは外部ライブラリを使わず、インラインSVGで表示します。

## 主な機能

- 依頼者区分に応じた入力と検証
- 写真・動画3点までの一時プレビュー
- 設定可能なルールベース概算
- 現地調査費と正式依頼時の差し引き条件の明示・同意
- 下書き、相談履歴、編集、複製、削除、対応状況、コピー、印刷
- 保存データのファイル書き出し・読み戻し
- 運営会社、ロゴ、料金、税区分、色、案内文の設定
- 固定受付メール宛ての問診メール作成と、対応端末での画像・動画共有

## ファイル構成

`index.html` が入口です。`css/` は変数・リセット・レイアウト・部品・レスポンシブに分割し、`js/` は画面、状態、検証、概算、料金、保存、設定、出力に分割しています。実装判断は `DESIGN.md` に記載しています。

## GitHub Pages

このディレクトリをリポジトリの公開ルートにするか、内容を `docs/` に配置し、Repository Settings → Pagesで対象ブランチとフォルダを選択します。すべて相対URLのためビルドは不要です。

GitHub Pagesへの初回公開は画面確認用です。個人情報と添付ファイルを扱う正式運用は、受付バックエンドと管理者認証の実装・検証後に開始します。

## 設定変更

ヘッダー右上の「設定」を開きます。会社名、ロゴ、現地調査費、正式依頼時の差し引き額、税込・税別、概算価格帯を変更して「設定を保存」します。初期値へ戻す場合は同画面のボタンを使います。

## 保存仕様と個人情報

案件、下書き、設定は利用中ブラウザのlocalStorageだけに保存されます。写真・動画の実データは保存も送信もされず、再読み込み後は件数情報だけが残ります。共用端末では案件を保存しないでください。ブラウザデータを削除すると案件も消えます。

履歴画面の「保存データをファイルに書き出す」で、パソコン・スマホのダウンロードフォルダへ保管できます。「保存ファイルから戻す」で追加読み込みできます。内部ファイル形式はJSONです。個別削除または全削除は確認後に実行され、元に戻せません。

## 業者設定のパスコード

ヘッダーの「業者設定」はパスコード入力後に開きます。初期パスコードは `1234` です。設定画面内で4〜8桁の数字へ変更できます。これは一般のお客様による誤操作を防ぐ簡易保護です。静的Webアプリのため、本格的なアクセス制御ではありません。機密情報を管理する場合は、将来の管理者ログイン機能へ移行してください。

## 複数症状と印刷

症状は1つ以上、複数選択できます。概算は価格帯が最も高い症状を主症状とし、2件目以降には設定可能な追加確認費を反映します。保存完了、保存履歴、相談詳細の「印刷」からブラウザの印刷画面を開けます。

## 試験メール送信

初期の問診受付メールは `makishi0520@gmail.com` です。保存完了画面の「問診内容をメールで送る」を押すと、端末のメールアプリへ宛先、件名、問診本文を渡します。メールは自動送信されず、お客様が内容を確認して送信操作を行います。

ブラウザの制約により、`mailto:` では写真・動画を自動添付できません。添付ファイルがある場合はメール画面で追加するか、「画像・動画をメール・LINEで共有」から端末の共有機能を利用します。共有機能では送信先が固定されないため、画面に表示された受付メールを確認してください。本番でワンタップ自動送信を実現する場合は、Supabase Storageとメール通知用サーバー処理へ移行します。

### 送信先を変更したお客様用URL

業者設定で会社情報、受付メール、料金、表示内容、ロゴ画像を入力し、「設定を保存してURLをコピー」を押します。コピーされたURLをホームページ、メール、QRコードなどでお客様へ案内すると、URLを発行した時点の公開設定がお客様の画面へ反映されます。ロゴ画像は64×64pxのWebP形式へ自動圧縮し、共有上限内の場合だけURLへ含めます。元画像はPNG・JPEG・WebP形式、1MB以下です。業者用パスコードは端末内だけに保存され、URLには含まれません。

会社情報、送信先、料金、表示内容を変更した場合は、新しいURLを案内し直してください。古いURLには発行時点の設定が残ります。URLの公開設定は利用者も確認・変更できるため、機密情報や管理用設定には使用しません。無効なメールを含むURLでは、既定メールへ誤送信せず送信操作を停止します。以前に発行した送信先メールだけのURLも引き続き利用できます。

設定画面では長い実URLを通常表示しません。「メール用リンクをコピー」を使うと、対応するメールアプリには「排煙窓事前チェックを開く」という短い表示のリンクとして貼り付けられます。端末がHTML形式のクリップボードへ対応していない場合は、案内文と実URLをコピーします。

「LINE用QR画像を作成」では、会社名・案内文・公開設定を含むQRコードを1枚の縦長PNG画像としてブラウザ内で生成します。「LINEなどへ送る」を押して端末の共有画面からLINEを選ぶため、トーク画面へ長い実URLを貼り付けません。QR案内画像はクリップボードへのコピー、PNG保存にも対応します。設定やロゴによりリンクがQRコードの容量を超える場合は発行を停止し、案内文またはロゴの見直しを求めます。QR生成には固定バージョンの `qrcode-generator` を使用し、設定情報を外部のQR作成サービスへ送信しません。

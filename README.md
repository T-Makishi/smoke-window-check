# 排煙窓かんたん事前チェック

排煙窓の修理・点検前に、お客様が問診と写真・動画を選択し、修理費用の目安と現地調査費の条件を確認できるWebアプリです。

公開先はCloudflare Pagesです。会社情報・料金・利用期限はCloudflare D1へ保存し、業者設定APIはCloudflare Accessのメール確認で保護します。問診内容、写真、動画は従来どおりお客様の端末内だけで扱い、Cloudflareへ保存しません。構成と安全上の判断は `CLOUDFLARE_DESIGN.md` を参照してください。

UIはデザインA「信頼感・安心感のあるベーシックデザイン」を採用しています。ダークグリーン、ネイビー、白を基調とし、料金案内だけを淡い黄色で明示しています。最大表示幅は480pxで、iPhone 13と320px幅へ対応しています。主要アイコンは外部ライブラリを使わず、インラインSVGで表示します。

## 主な機能

- 依頼者区分に応じた入力と検証
- 写真・動画3点までの一時プレビュー
- 設定可能なルールベース概算
- 現地調査費と正式依頼時の差し引き条件の明示・同意
- 下書き、相談履歴、編集、複製、削除、対応状況、コピー、印刷
- 保存データのファイル書き出し・読み戻し
- 運営会社、ロゴ、料金、税区分、色、案内文の設定
- 固定受付メール宛ての問診メール作成と、対応端末での画像・動画共有
- 業者ごとの短いお客様用URL
- 7日・14日・30日の試験利用期限
- 期限終了・利用停止時の公開画面制御
- メール確認コードで保護した業者設定
- サービス運営者用の業者登録・期限管理画面

## ファイル構成

`index.html` が入口です。`css/` は変数・リセット・レイアウト・部品・レスポンシブに分割し、`js/` は画面、状態、検証、概算、料金、保存、設定、出力に分割しています。実装判断は `DESIGN.md` に記載しています。

`main` へのpush後、Cloudflare Pagesが自動でデプロイします。GitHub PagesはCloudflare版の期限確認と認証を検証した後に停止します。

## Cloudflare初期設定

1. D1データベース `smoke-window-check-db` を作成する。
2. D1コンソールで `schema.sql` を実行する。
3. PagesプロジェクトのSettings → Bindingsで、D1を変数名 `APP_DB` に割り当てる。
4. Settings → Variables and Secretsで `SERVICE_ADMIN_EMAIL=makishi0520@gmail.com` を設定する。
5. 設定反映後に本番デプロイを再実行する。
6. Cloudflare Zero TrustでOne-time PINを有効にする。
7. `smoke-window-check.pages.dev/api/vendor/*` と `smoke-window-check.pages.dev/api/service/*` をAccessで保護し、セッション時間を24時間にする。
8. Access通過後もAPI側で認証メールを照合するため、業者はD1に登録したメール、サービス運営者は `SERVICE_ADMIN_EMAIL` だけが操作できる。
9. Pages Functionsの無料枠到達時の動作をFail closedにする。

設定後は `https://smoke-window-check.pages.dev/service.html` を開き、業者名、業者ログイン用メール、7日・14日・30日の期間を入力して登録します。発行された `?t=...` のURLをお客様へ案内します。

## 設定変更

お客様用URLを開き、ヘッダー右上の「業者ログイン」を押します。D1に登録した業者メールへCloudflareから確認コードが届きます。認証後、会社名、受付メール、ロゴ、現地調査費、正式依頼時の差し引き額、税込・税別、概算価格帯を変更して保存します。

## 保存仕様と個人情報

案件、下書き、設定は利用中ブラウザのlocalStorageだけに保存されます。写真・動画の実データは保存も送信もされず、再読み込み後は件数情報だけが残ります。共用端末では案件を保存しないでください。ブラウザデータを削除すると案件も消えます。

履歴画面の「保存データをファイルに書き出す」で、パソコン・スマホのダウンロードフォルダへ保管できます。「保存ファイルから戻す」で追加読み込みできます。内部ファイル形式はJSONです。個別削除または全削除は確認後に実行され、元に戻せません。

## 旧GitHub Pagesとの互換性

`recipient` と `cfg` を含む従来URLは移行確認中のみ利用できます。従来の業者設定は端末内パスコード `1234` の簡易保護です。Cloudflareの `?t=...` URLでは端末内パスコードを使用せず、メール確認を使用します。移行検証後はGitHub Pagesを停止し、従来URLを正式運用へ使用しません。

## 複数症状と印刷

症状は1つ以上、複数選択できます。概算は価格帯が最も高い症状を主症状とし、2件目以降には設定可能な追加確認費を反映します。保存完了、保存履歴、相談詳細の「印刷」からブラウザの印刷画面を開けます。

## 試験メール送信

初期の問診受付メールは `makishi0520@gmail.com` です。保存完了画面の「問診内容をメールで送る」を押すと、端末のメールアプリへ宛先、件名、問診本文を渡します。メールは自動送信されず、お客様が内容を確認して送信操作を行います。

ブラウザの制約により、`mailto:` では写真・動画を自動添付できません。添付ファイルがある場合はメール画面で追加するか、「画像・動画をメール・LINEで共有」から端末の共有機能を利用します。共有機能では送信先が固定されないため、画面に表示された受付メールを確認してください。本番でワンタップ自動送信を実現する場合は、Supabase Storageとメール通知用サーバー処理へ移行します。

### お客様用URL

Cloudflare版のお客様用URLは `https://smoke-window-check.pages.dev/?t=<発行ID>` です。会社情報、受付メール、料金、表示内容、圧縮済みロゴはD1から取得するため、設定変更後も同じ短いURLを継続利用できます。元ロゴ画像はPNG・JPEG・WebP形式、1MB以下です。

お客様がURLを開くたびにD1の利用状態を確認します。期限終了、利用停止、登録なし、D1接続不可の場合は問診を表示しません。お客様の問診データや写真・動画はこの確認処理で送信されません。

設定画面では長い実URLを通常表示しません。「メール用リンクをコピー」を使うと、対応するメールアプリには「排煙窓事前チェックを開く」という短い表示のリンクとして貼り付けられます。端末がHTML形式のクリップボードへ対応していない場合は、案内文と実URLをコピーします。

「LINE用QR画像を作成」では、会社名・案内文・公開設定を含むQRコードを1枚の縦長PNG画像としてブラウザ内で生成します。「LINEなどへ送る」を押して端末の共有画面からLINEを選ぶため、トーク画面へ長い実URLを貼り付けません。QR案内画像はクリップボードへのコピー、PNG保存にも対応します。設定やロゴによりリンクがQRコードの容量を超える場合は発行を停止し、案内文またはロゴの見直しを求めます。QR生成には固定バージョンの `qrcode-generator` を使用し、設定情報を外部のQR作成サービスへ送信しません。
